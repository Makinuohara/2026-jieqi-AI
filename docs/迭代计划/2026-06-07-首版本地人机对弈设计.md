# 首版本地人机对弈设计

## 目标

第一版提供可实际操作的本地真人对随机人工智能对弈。真人执红先行，人工智能执黑。
玩家能够选择棋子、查看合法落点、完成走子和吃子；人工智能随后自动回应。任一方将帅
被吃后立即结束，并可重新开始。

## 本轮范围

本轮实现：

- 七类棋子的基础移动与吃子规则。
- 马腿、象眼、炮架和将帅照面限制。
- 明士、明象允许过河和离开九宫。
- 暗子按所在初始位置对应的虚拟棋种移动。
- 暗子第一次合法移动或吃子后，由裁判从剩余类型池随机翻开。
- 合法走法生成、走法执行、轮流行棋和吃将终局。
- 真人执红、随机人工智能执黑的本地对局。
- 中文棋盘、合法落点提示、走法记录、重新开始和认输。
- 人工智能在后台线程思考，避免阻塞 JavaFX 界面。

本轮暂不实现：

- 将军后必须应将。
- 80 个半回合无吃子判和。
- 长将、长捉和兵卒例外。
- 联网匹配、服务器裁决和棋谱持久化。
- 贪心人工智能和 Expectiminimax 的完整搜索。

## 核心结构

### 完整对局状态

`GameState` 除棋盘、当前回合和终局状态外，还保存红黑双方剩余暗子类型池。每次暗子
首次移动或吃子后，规则引擎只从该方池中抽取一个类型，并将池同步更新到新状态。

### 规则引擎

`GameEngine` 提供两个入口：

- `legalMoves(GameState)`：生成当前行棋方全部合法走法。
- `apply(GameState, Move)`：校验并执行走法，产生新状态和事件。

走法生成与执行共用同一移动规则，避免界面提示合法但执行时被拒绝。

### 本地对局服务

新增与 JavaFX 无关的 `LocalHumanVsAiGame`：

- 保存当前 `GameState`。
- 接收真人走法。
- 根据规则引擎生成 `PlayerView`。
- 调用随机人工智能选择黑方走法。
- 保存公开的走法记录和终局结果。

该服务可以独立测试，不依赖图形界面线程。

### 图形界面

JavaFX 首页保留四个入口，但本轮只接通“真人对人工智能”。对局页面显示中文棋子、
当前回合、对局状态、走法记录和控制按钮。点击己方棋子后高亮合法落点；点击合法落点
后执行真人走法，再通过后台任务触发人工智能回合。

## 规则细节

- 车沿横线或纵线移动，中间不得有棋子。
- 马走日，马腿位置不得有棋子。
- 炮不吃子时中间不得有棋子；吃子时中间必须恰有一个炮架。
- 兵卒未过河只能向前一步，过河后可向前或横向一步，不得后退。
- 将帅每次横向或纵向一步；不能在走后与对方将帅直接照面。
- 士每次斜向一步，不限制九宫。
- 象每次斜向两步，象眼不得有棋子，不限制过河。
- 暗子的 `virtualType` 决定首次移动规则，翻开后的 `actualType` 决定后续规则。
- 吃掉对方将帅后，状态立即变为红胜或黑胜。

## 错误处理

- 非当前行棋方、空起点、己方棋子占据终点和非法移动均返回明确错误。
- 已结束对局拒绝继续走子。
- 人工智能无合法走法时，对局结束并判对方获胜。
- JavaFX 后台任务失败时恢复界面可操作状态并显示中文错误信息。

## 测试与验收

- 每类棋子至少覆盖一个合法移动和一个非法移动。
- 单独覆盖马腿、象眼、炮架、将帅照面和兵卒过河。
- 验证暗子按虚拟类型移动、首次移动后翻开且暗子池减少。
- 验证吃将立即终局。
- 验证随机人工智能只从合法走法中选择。
- 验证一局真人对人工智能可以连续推进多个回合。
- `.\mvnw.cmd test` 全部通过。
- `.\run-gui.cmd` 能打开中文界面，真人可选择棋子并完成至少一次走子，人工智能能自动回应。
