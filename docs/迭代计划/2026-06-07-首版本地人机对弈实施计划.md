# 首版本地人机对弈实施计划

> 执行时按测试驱动方式逐项完成，每个任务结束后运行对应模块测试。

## 目标

在现有七模块框架上完成可操作的本地真人对随机人工智能对弈闭环。

## 技术路线

规则引擎成为合法走法的唯一来源，`GameState` 保存完整暗子池状态，本地对局服务编排
真人与人工智能回合，JavaFX 只负责显示和用户输入。

## 任务一：扩展完整对局状态（已完成）

- 修改 `jieqi-model`，增加不可变的暗子池快照类型。
- 将红黑双方暗子池加入 `GameState`。
- 扩展 `Board` 的移动、翻开和测试构造能力。
- 增加初始状态、池数量和不可变更新测试。

验收命令：

```powershell
.\mvnw.cmd -pl jieqi-model test
```

## 任务二：实现基础走法规则（已完成）

- 新增统一的棋子移动规则组件。
- 实现车、马、炮、兵卒、将帅、士和象。
- 实现马腿、象眼、炮架、过河和将帅照面。
- 扩展 `GameEngine` 的 `legalMoves` 接口。
- 用参数化和边界测试覆盖规则。

验收命令：

```powershell
.\mvnw.cmd -pl jieqi-rules -am test
```

## 任务三：实现状态推进与翻子（已完成）

- 用正式规则引擎替换 `SkeletonGameEngine`。
- 执行走法时处理吃子、回合切换和无吃子计数。
- 暗子首次移动后从对应池中抽取类型并发出翻子事件。
- 吃掉将帅后立即设置胜方状态。
- 增加翻子、吃将和已结束对局测试。

验收命令：

```powershell
.\mvnw.cmd -pl jieqi-rules -am test
```

## 任务四：实现本地人机对局服务（已完成）

- 在 `jieqi-gui` 中新增不依赖 JavaFX 控件的本地对局服务。
- 真人固定执红，随机人工智能固定执黑。
- 服务生成玩家视图、提交真人走法并执行人工智能回应。
- 保存中文走法记录和对局状态。
- 增加多回合推进及人工智能合法性测试。

验收命令：

```powershell
.\mvnw.cmd -pl jieqi-gui -am test
```

## 任务五：接通中文 JavaFX 棋盘（已完成）

- 将棋盘格绑定真实 `Position` 和 `Piece`。

- 点击棋子后显示合法落点，点击落点后提交走法。
- 人工智能通过 JavaFX 后台任务执行。
- 增加重新开始、认输、状态栏和走法记录。
